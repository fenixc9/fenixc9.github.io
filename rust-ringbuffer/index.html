用Rust实现一个无锁RingBuffer

RingBuffer在网络通信，消息队列中都有广泛的应用。今天，我们尝试在AI的辅助下，逐步从0实现一个好用的RingBuffer。

RingBuffer是一个循环数组结构，包含两个逻辑指针。维基百科中更倾向于称之为”Circular buffer”，环形缓冲区：

*圆形缓冲区（circular buffer），也称作圆形队列（circular queue），循环缓冲区（cyclic buffer），环形缓冲区（ring buffer），是一种用于表示一个固定尺寸、头尾相连的缓冲区的数据结构，适合缓存数据流。

RingBuffer中不需要调整其容量，生成后则后续只移动其读写指针。

它似乎可以在单线程环境下正确运行，但不是我们想要的多线程无锁版。

这里由于我们需要使用take从Option中取出数据。然而：

  1. Option，take需要接受一个&mut self
  2. 这就导致我们的pop和push也必须是&mut self
  3. 然而，当我们想把我们写好的RingBuffer通过Arc分发到各个线程时会发现，Arc并不能获取可变引用，Arc只实现了Deref，并没有实现DerefMut。
  4. 这就导致必须使用Mutex才能解决我们的问题，这就违反了我们需求。

因此，如果这里想实现无锁，我们仍需要尝试使用别的数据容器。

一种可行的方式是为我们的数据容器包一层UnsafeCell，顺便把usize改为AtomicUsize

我们首先面向SPSC场景，单线程生产则不需要write的SeqCst，因为根本不会发生写冲突，写线程始终能读到最新的write，同样的道理，消费者
则始终能获取到最新的read点位。而生产者的read和消费者的write则由对方的Release保证。

这种SPSC情况下，我们就能使用&self 代理 &mut self。

同时我们保留一个哨兵位，即用于区分RingBuffer为空和满两种情况：read==wirte为空，read==write+1 则为满。

再来点小优化，比如我们使用取模效率相对较低，我们将其改造为强制使用2的n次幂下的位运算。

我们将不会使用裸指针，因为我们在RingBuffer初始化时会将内存分配好，并且始终不会move，我们可以使用NonNull。

需要注意的是，由于我使用指针代替了Vec，这意味着当我们Drop RingBuffer时，我们需要手动调用T的Drop函数，并释放内存。
